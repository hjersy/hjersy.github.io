FlyData Wizard

Brief

At FlyData, we know to focus on uncovering data to make business decisions is already a tough job, we want to help customer to manage the entire process of data integration and no need to maintain their own replication service. We help our customer to integrate data from different data source to Amazon Redshift data warehouse, so they can analyze production data in near real-time without changing back-end infrastructure.

Why we need wizard and the scope

To enhance the first time user experience and reduce all the hassle of setup steps to start the integration in a short time, we need to design a flow to smooth all configuration between FlyData Console and AWS Console, and remove the frustrated points as mush as possible when user try to sync their database to AWS Redshift through our service with a short time. Therefore, we design the Wizard and the goal of FlyData Wizard is to let user do the minimum setup by telling us where the data destination is and where we should pull the data source from, then they can just forget about the integration part and really focus on data analysis to develop their business.

The issue and what the old flow is

Before, when user signed up the trial, they needed to have 3 browser tabs open to read through the getting started guide, copy the instance information from AWS console, then paste to FlyData Console. But imagine if you are one of them, you need to remember the step on the started guide, and try to find the information on AWS console then the fill up input field in FlyData Console, back and forth to configure your AWS Instance by repeating the same step, then realized you done something wrong at the very end of the step. How frustrated would you be?

With the current flow, tech sales need to schedule a meeting with user and guide them step by step to finish the initial setup, if user has questions, sales would invite a support engineer to troubleshooting together and it might take 1-2 business days to complete the setup process by back and forth.

Interaction

The Impact

The console wizard has a positive impact on on first-time user on boarding experience, it reduce the configuration bumps and helps user trouble shooting in the real time with automated validation which require minimum resource of support engineers involved. Further, we integrate with Mixpanel and Segment to have better usage insight data to measure the funnel to know if user stuck on certain page to determine if there’s any issue, our support/sales engineer will approach to help right away. It also rude some user’s fear of talking to sales before they start the trial.

- User would able to start trial setup within mins, instead of 1 day to wait the link send by our sales.

- User can setup the first application within 40-50 mins, it smooth the experience by reducing from 3-4 hours

- It save sales/support engineer almost 2 days to assist 1 user on setting the first application

- The automation helps our engineers to focus on develop tasks instead of trouble shooting with the validation
