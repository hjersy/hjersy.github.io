FlyData Console Replacement Rule

Brief

The Replacement Rule feature allows a FlyData user to do customize their “Transform” layer of FlyData’s ETL product. The user is allowed to customize string replacement to do some simple pre-processing of their data or to simply change some text to fit Amazon Redshift’s data formats.

For example, you may want to remove or hide data before it’s sent to Amazon Redshift. Here, you can define a replacement rule for specific columns so that you replace/remove all the sensitive data before loading them into Amazon Redshift.

Why I want to design a GUI instead of JSON format

Even though JSON is already a standardized way of representing data, it is still hard to expect a customer to have perfect JSON and have an understanding of their result. I found that JSON had some flaws where:

  • JSON is fragile and it’s not always obvious where issues are
  • Given a format, it may be easy to have a mistake in the syntax and accidentally modify the wrong data.
  • The repetition of field names feel redundant and unnecessary for a user to have to worry about. For example, the resulting JSON form would look something like this:
[
    { "table" : "users”,
        "column" : "user_id”,
        "replace_rules" : [
            {"^-$" : null},
            {"#$" : “"}
        ]
    },
    { "table" : "orders”,
        "column" : "name”,
        "replace_rules" : [
            {"^(.*)$":"\\1!”}}
        ]
    }
]

There were also specific things I had to keep in mind that had to be conveyed to the user. I had to make sure to convey each part of the JSON such as Table, Column, Replace Rule with its find/replace with functionality. Also, I had to convey that each column from a table can have multiple rules. However, I had to make sure the user followed a “Table” -> “Column” -> “Replace Rules” flow.

Program logic

  • To figure out the pattern, the necessary object/column, which are “Table”, “Column” and “Replace_rule”. Also in “Replace_rule”, it includes 2 strings “find” and “replace with”
  • One table can have different column, and each column can have multiple rules.
  • The hierarchy: “Table”->”Column”->”Replace_rules”

Good to have

  • Auto complete form for “table”, “Column” if the value is exist
  • Index “Table” and sort by "Alphabet”
  • Auto Convert JSON format to table view when uploading JSON
  • Export as JSON format file

Interaction

Conclusion

This Replacement Rule UI provides 2-way edit methods which is traditional for both JSON codebase format and GUI. It gives the user flexibility to easily manage it in a more natural efficient way. The user can also import their CSV/JSON format to the UI and we also transform those files into GUI. As a GUI system for Replacement Rule UI, it also offers an easier way for the user to confirm the rules have been input correctly because removing the syntax reduces the review time.
